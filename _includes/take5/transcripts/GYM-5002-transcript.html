In today’s Take 5 you’re going to take a plain heading as seen here and add a “knockout” effect, which means you can take any image you’d like and make it appear inside the outline of your text, the example you’ll be using is an image of clouds and the text you’ll be using are the words “The Cloud”.

This is the project on Codepen you’ll be working with and here is the URL which you can also find linked within the Resources section for this video.

In the HTML, there’s a Heading 1 with the class of “clip” and your content. In the CSS there is an h1 style with the font-family, size and color.

There’s also this section here which is a CSS Feature Query. If you haven’t familiar with this syntax, don’t worry, I’ll explain in more detail shortly, for now, just note the class selector named “clip” located here is empty. To save you some typing, I put the first two styles right here in the comments, all you need to do is cut both of them and then paste them into the clip class like so: 


@supports (-webkit-background-clip: text) and (-webkit-text-fill-color: transparent) {
 
  .clip{
	background-image: url('https://s3-us-west-2.amazonaws.com/s.cdpn.io/76301/clouds_800x200.jpg');
	background-repeat: no-repeat;
  }
 
}

So you’ll see a background image of clouds appear behind your heading, this image is 800 pixels wide by 200 pixels high and right now the whole thing looks terrible, but now you’ll use the outline of your text to clip the image. So go ahead and type: 

    
-webkit-background-clip :text;

Your background image disappears, which means clip is working, but the text is still being affected by your h1 color, so you’ll now force this color to be transparent,so type: 
  
-webkit-text-fill-color: transparent;

That’s the effect! (PAUSE) The outline of your text is now clipping the image and you could use this as is: However, I promised you a quick explanation: this clip class is nested inside a CSS Feature Query and what this syntax does is test a browser to see if it supports these two features 

(-webkit-background-clip: text) and (-webkit-text-fill-color: transparent)
. Both of these are actually well supported in most modern browsers. However there are some exceptions, and older browsers definitely don’t support this CSS. So it’s important you have a fallback style and in this case the fallback is that the first h1 style is what those users see. So, no clipping effect, in fact no image: just the plain h1 style that we saw at the beginning.

Now, if you want to learn more about how CSS feature queries actually work, be sure to check out the Resources section for this video.
