Blend modes are something you might be familiar with from Photoshop where two layers can be blended together to create a new image. This method is similar but uses plain old HTML and CSS, no software required, you dig?

This is the project on Codepen you’ll be working with and my workspace looks different because I’m using the DuoTone Dark theme, which our main daddy-o at Codepen Chris Coyier has conveniently added for us and here is the URL so you can follow along. You can also find the project link in the Resources section for this video.

Unlike Photoshop, in CSS you don’t have layers, so you’ll see the image is nested in a div with the class of “duotone”. The styles for that class and the image declare the width of the div to 600 x 600 and the image to 100% width.

I’ve grouped these styles together for you: duotone::before and duotone::after. These are pseudo elements, special selectors that allow you to insert content into a page using CSS. Typically you would insert content inside these quotes but you’re leaving it blank and the rest of the code fills the entire width and height of your div with an empty style that you can add a color to. The first color will be the highlight of your duotone. So type the following and pause the video if you have to, there ain’t no rush today


.duotone::after {
	background-color: hsla(164, 50%, 50%, 1);
  }

So check it out in hsla the value 164 is a blue-green hue, the first 50% is the saturation, the second 50% is the luminance and this 1 for transparency or alpha. The color covers your image, now you’ll blend it by typing:


mix-blend-mode: darken;

Can you dig it, man, that's crazy! It’s all about the maths my friends: The darken blend mode checks to see if the pixels in your color are darker than the ones in the image. If they *are* darker: then they stay.

So try changing the effect with the luminance value. Try 20% luminance and your color affects more of the highlights, now try 80% and your color affects less of the highlights. Go ahead the luminance back to the original 50%, you can always mess around with it later.

I want you now to turn this effect off temporarily by cutting the last bit of commenting code and pasting it at the end of this line. Now you’ll add a second color so type the following:


.duotone::before {
	background-color: hsla(291, 50%, 50%, 1);
  }

291 is a purple-pink hue, the saturation and lightness values are both 50%, and the alpha remains 1. Now type


mix-blend-mode: lighten;

This is the opposite effect of the first example. Here, all of the pixels in your color that are lighter than the background are retained which means the color primarily affects the shadows.

Now what you’re about to do is really something else, man, cut and paste this commenting code back to the first line and you’ll see both colors and blend modes interacting with each other. There’s your duotone and now you’re swinging.

Now you can just play around and see what happens, it’s like Jazz baby, a little improvisation never hurt noone: so try changing the lightness value of your highlight from 50 to 76%. 


background-color: hsla(164, 50%, 76%, 1);

Out of sight right?, if that’s too much you can bring the lightness of your shadows down to 25%, for a more subtle duotone. It’s all up to you, just do your thing!

One more thing you need to do which is kind of a drag: we gotta add some browser support for browsers that don’t support this cool effect. Type the following line above your highlight code


@supports (mix-blend-mode: darken) {

Be sure to add a closing curly brace at the very end

For the second style add


@supports (mix-blend-mode: lighten) { and be sure to add the closing curly brace.

What it does is give browsers that don’t support the CSS mix-blend modes the original grayscale image, which is a bit of a bummer but at least they will get something, you dig?
